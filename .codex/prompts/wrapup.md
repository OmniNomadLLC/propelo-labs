# Propelo Labs — Codex Wrap-Up Protocol

## Close With Discipline
- End each work session with a structured review, not just "done".
- Re-read key docs as needed to confirm alignment:
  1. `docs/PROJECT KICK-OFF.md`
  2. `docs/PROJECT GPT INSTRUCTIONS — MEMORY & QUALITY PROTOCOL.md`
  3. `docs/PROJECT MEMORY — AI Build Platform.md`

## Verification Checklist
- Scope control upheld? No unrelated edits?
- Frontend-only rule honored unless new authorization was given?
- Deliverables match the Phase 1 wedge (landing page, app shell, five core screens)?
- Risks, blockers, and edge cases documented?
- Handoff ready for the next builder?

## Final Summary Requirements
Provide a concise closing packet that includes:
1. What changed and why.
2. Files touched (paths) and notable diffs.
3. What was intentionally not changed or deferred.
4. Any mocks, placeholders, or stubs introduced.
5. Risks, open questions, or follow-up tasks.
6. Suggested next step aligned with the roadmap.

## Project Snapshot
- Append a "Project Snapshot" section ready for Project Memory using this exact scaffold:

  ### Project Snapshot – [DATE]

  **Context**  
  What this step focused on.

  **Decisions locked in**  
  What is now fixed.

  **New insights**  
  What became clearer.

  **What is explicitly out of scope**  
  What is intentionally not part of this step.

  **Product / UX consequences**  
  What this means for product design or structure.

  **Next recommended step**  
  What should happen next.

- Keep the snapshot tight and continuity-ready so the next session can resume without re-discovery.

Call this prompt with `wrapup.md` to end a Codex session.
