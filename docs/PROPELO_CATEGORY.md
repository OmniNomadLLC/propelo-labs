# Propelo Category Strategy

## 1. The Problem
Most tools optimize execution, not architecture. Notion captures knowledge, Jira manages tasks, Linear orchestrates sprints, Figma designs interfaces—but none help teams design systems. Decisions vanish, risks stay hidden, execution collapses without architectural clarity.

## 2. The Propelo Category
Propelo introduces the **Architecture Design Platform** category. Instead of jumping straight into execution, teams design systems step by step.

## 3. Category Definition
An Architecture Design Platform provides:
- structured system thinking
- architecture decision tracking
- risk mapping
- system blueprint modeling
- execution planning linked to architecture
- architecture memory over time

## 4. The Propelo Stack
Propelo combines:
- Architecture Objects
- Architecture Graph
- Reasoning Engine
- Data Flywheel
- Architecture Workspace
Together they form an operating system for system design.

## 5. Comparison With Existing Tools
- Figma → UI architecture
- GitHub → code architecture
- **Propelo → system architecture**

## 6. Why This Category Matters
Complex systems require structured design. Without architectural thinking, teams face hidden risks, lost decisions, and execution chaos. Propelo provides a system for designing systems.

## 7. Long-Term Vision
Propelo evolves into a platform that understands how systems are designed—suggesting architecture patterns, detecting design risks, generating blueprints, guiding execution planning.

## Final Principle
**Propelo defines the category of architecture design platforms.**
