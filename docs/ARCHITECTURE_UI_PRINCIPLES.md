# Propelo Architecture UI Principles

## 1. Core Principle
The UI exists to support architecture thinking. Every screen must reinforce the pipeline: Mission → Decisions → Risks → Blueprint → Tasks → Wrap Snapshot.

## 2. Structured Interfaces
Propelo never presents a blank canvas. Users interact with structured objects—decision cards, risk nodes, blueprint phases, execution tasks, wrap snapshots—so the interface guides structured reasoning.

## 3. Calm System Design
The workspace should feel calm and analytical: low visual noise, clear hierarchy, structured layouts, minimal decoration. The UI should read like an operating console, not a marketing page.

## 4. Object-Centered Design
All interactions revolve around architecture objects. Users manipulate decisions, risks, blueprints, tasks, and snapshots. The interface must keep these objects visible and connected.

## 5. Progressive Architecture
Complexity reveals progressively: Mission (definition) → Decisions (reasoning/tradeoffs) → Blueprint (system structure) → Tasks (execution) → Wrap (architecture memory). Each screen deepens architectural fidelity.

## 6. Architecture Context
Context must stay visible: decisions impacting blueprint, risks linked to decisions, tasks tied to blueprint phases, snapshots capturing system state. Every surface reminds users where an object fits in the system.

## 7. Avoid Productivity Tool Patterns
Reject generic patterns—blank note pages, infinite documents, kanban boards without architecture context, chat-only workflows, arbitrary dashboards. If it doesn’t reinforce structured architecture, it doesn’t ship.

## 8. System Thinking UX
Interfaces should teach users to think in systems: impact indicators, architecture flows, risk connections, phase boundaries, readiness states. Show how objects influence one another.

## 9. Workspace Consistency
All workspace areas must feel like one operating system: Brain (reasoning), Scope (architecture), Tasks (execution), Wrap (memory). Consistency maintains trust and clarity.

## Final Principle
**The Propelo interface is designed to support architectural thinking.**
