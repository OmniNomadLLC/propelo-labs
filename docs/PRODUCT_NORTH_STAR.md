# Propelo Product North Star

## 1. Purpose
This document defines the long-term direction for Propelo. It keeps every roadmap, feature request, and design decision anchored to the architecture-first mission so the product never drifts into generic productivity tooling.

## 2. Core Idea
Propelo helps users design systems. It shepherds every problem through the pipeline:

Problem → Architecture → Execution → Memory

Propelo’s value compounds when each stage is structured, reviewable, and deeply connected to the others.

## 3. The Propelo Method
1. **Define mission** – capture the problem, constraints, and success signals.
2. **Identify architectural decisions** – surface assumptions, risks, and guardrails.
3. **Design blueprint** – map phases, architecture flows, and smallest useful versions.
4. **Translate into execution tasks** – assign proof-of-work tasks aligned to phases/modes.
5. **Capture project memory** – wrap snapshots keep decisions and outcomes persistent.

## 4. Product Category
Propelo is an **Architecture Design Platform.**
- Figma → UI architecture
- Linear → engineering workflow
- Notion → knowledge systems
- **Propelo → system architecture**

## 5. Architecture Engine
The workspace is a structured engine that transforms inputs into system outputs:

Mission → Decisions + Risks → Blueprint → Tasks → Wrap Snapshot

Each screen reinforces the pipeline to ensure architectural integrity.

## 6. Architecture Memory
Propelo stores architecture data over time. The system records:
- decisions
- risks
- blueprint versions
- execution tasks
- session snapshots

Together they form an evolving architecture dataset the product can reason over.

## 7. Viral Growth Loop
Propelo spreads through the artifacts it creates:
- sharable blueprint links
- project story pages that narrate the build
- system design snapshots for stakeholders
Each artifact showcases the structured thinking, inviting others into the workspace.

## 8. UX Philosophy
The UI must feel calm, analytical, structured, and professional. Avoid generic productivity UI, feature overload, or workspace chaos. Every interaction should reinforce clarity, order, and architectural thinking.

## 9. Final Principle
**Propelo is a thinking operating system for designing systems.**
