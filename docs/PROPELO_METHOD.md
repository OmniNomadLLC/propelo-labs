# The Propelo Method

## 1. The Problem With Unstructured Work
Most teams build in unstructured environments—chat threads, docs, kanban boards, endless meetings. These tools capture activity but not architecture. Decisions get lost, risks stay hidden, execution drifts into chaos.

## 2. The Propelo Solution
Propelo replaces chaos with a structured architecture workflow. Teams design systems step by step:

Mission → Decisions → Risks → Blueprint → Tasks → Wrap Snapshot

## 3. Step 1: Define the Mission
The mission captures the problem, constraints, and success criteria. It anchors all downstream architecture decisions.

## 4. Step 2: Record Architectural Decisions
Document major design choices—tech stack, system boundaries, integration approach, deployment strategy—with reasoning and confidence so the system understands tradeoffs.

## 5. Step 3: Map Risks
Every decision introduces risks. Propelo forces teams to log technical uncertainty, performance constraints, integration complexity, organizational dependencies, etc., so nothing hides in the margins.

## 6. Step 4: Design the Blueprint
Translate decisions and risks into the system blueprint. Phases/components describe how the architecture will unfold and connect structure to execution.

## 7. Step 5: Generate Execution Tasks
Blueprint phases turn into proof-of-work tasks. Each task carries architecture context so execution never detaches from design intent.

## 8. Step 6: Capture a Wrap Snapshot
Wrap snapshots freeze the architecture state (mission, decisions, risks, blueprint, tasks) at a point in time. They create the project’s architecture memory.

## 9. Continuous Architecture Thinking
Teams repeat the Propelo Method as systems evolve: new missions or sub-missions, fresh decisions, changing risks, evolving blueprints, updated tasks, new snapshots.

## 10. Why This Matters
The Propelo Method transforms chaotic work into a structured, repeatable system design process. Instead of scattered notes and disconnected tasks, teams build an architecture record that becomes valuable data over time.

## Final Principle
**The Propelo Method turns system design into a structured, repeatable process.**
