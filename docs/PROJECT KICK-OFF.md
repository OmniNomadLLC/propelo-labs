🟦 A) NEW PROJECT KICK-OFF — CHAT SUMMARY TEMPLATE
👉 Gebruik dit ALTIJD als eerste inhoud in een nieuw Project Memory document.
🔹 Chat Summary – [DATUM]
(Project Kick-off)

Context van de chat
Dit is de start van een nieuw project.
De chat had als doel het project te definiëren, het initiële doel te verwoorden
en de eerste kaders vast te leggen, met de expliciete erkenning dat dit doel
kan evolueren naarmate inzichten groeien.

Beslissingen die vastliggen
● Het Project Memory document is de enige bron van waarheid voor dit project.
● De Project GPT Instructions en geheugen-protocol worden ongewijzigd toegepast.
● Wijzigingen aan doelen, aannames of scope gebeuren uitsluitend via Chat Summaries.

Nieuwe inzichten
● Eerste inschatting van het probleem/domein.
● Eerste ideeën over mogelijke aanpak of richting.
● Bewustzijn dat deze inzichten voorlopig en onvolledig zijn.

Wat expliciet NIET de bedoeling is
● Geen definitieve conclusies trekken in deze fase.
● Geen impliciete aannames als feiten behandelen.
● Geen scope uitbreiden zonder expliciete beslissing.

Open vragen / volgende stappen
● Welke aannames moeten eerst gevalideerd worden?
● Welke informatie ontbreekt nog?
● Wat is de eerstvolgende concrete stap om te leren of te testen?

📌 Belangrijk
Dit is geen inhoudelijke beslissing, maar een formele startmarker.
Alles wat volgt bouwt hierop voort.

🟦 B) SOP — “ZO START IK ELK NIEUW PROJECT”
👉 Dit is je vaste werkwijze.
Sla dit desnoods op als checklist of README voor jezelf.

🧭 Doel van deze SOP
Deze SOP beschrijft hoe elk nieuw project wordt opgestart, zodat:
niets uit het verleden vergeten wordt
geen context verloren gaat
elk project reproduceerbaar, traceerbaar en schaalbaar blijft

🟩 STAP 1 — Project Memory aanmaken
Dupliceer 📁 PROJECT MEMORY – STANDARD TEMPLATE
Hernoem naar:
📁 PROJECT MEMORY – [Projectnaam]
Vul bovenaan in:
Project naam
Project groep (optioneel)
Startdatum
Laatste update
❌ Nog geen inhoud toevoegen
❌ Geen secties aanpassen (template is gefreezed)

🟩 STAP 2 — Project GPT Instructions NIET aanpassen
De Project GPT Instructions blijven identiek
Ze gelden automatisch voor dit nieuwe project
👉 Dit garandeert:
identiek geheugen-gedrag
identieke kwaliteitsnormen
geen interpretatieverschillen tussen projecten

🟩 STAP 3 — Eerste Chat Summary = Kick-off
Start een chat binnen het nieuwe project
Vraag:
“Maak een Project Kick-off Chat Summary”
Gebruik exact het Kick-off template (zie A)
Voeg deze toe onderaan het Project Memory document
Upload het document opnieuw
👉 Dit is het officiële beginpunt van het project.

🟩 STAP 4 — Initiële doelstelling invullen
Ga naar ALGEMENE PROJECTDOEL & DOEL-EVOLUTIE
Beschrijf:
het huidige doel
zonder zekerheid of belofte
als startpunt, niet als eindpunt
📌 Het doel mag:
breed zijn
vaag zijn
veranderen

🟩 STAP 5 — Aannames en scope expliciet houden
Gebruik vanaf nu altijd:
Vaste beslissingen → alleen als iets echt vastligt
Aannames & hypotheses → voor alles wat nog onzeker is
Buiten scope (voorlopig) → voor focus en rust
👉 Niets blijft impliciet.

🟩 STAP 6 — Werken = geheugen bijhouden
Vanaf dit punt geldt automatisch:
Elke belangrijke chat → nieuwe Chat Summary
Elke wijziging → expliciet gedocumenteerd
Elke twijfel → liever vastleggen dan vergeten
👉 Het systeem vangt fouten op vóór ze ontstaan.

🧠 Kernprincipe (onthouden)
Nieuw project = zelfde regels, nieuw geheugen.
Gedrag blijft constant.
Inhoud evolueert.

🔐 Git Setup (Multi-Account – Short)
Deze machine gebruikt meerdere GitHub accounts (werk + privé).


Altijd SSH gebruiken, nooit HTTPS (vermijdt account-conflicten).


git init
git branch -M main
git remote add origin git@github.com:<ACCOUNT>/<REPO>.git
git remote -v
ssh -T git@github.com
git add -A
git commit -m "chore: initial setup"
git push -u origin main

Rules
.env nooit committen


.env.example altijd committen


Werk & privé repos altijd gescheiden SSH identities


Bij repository not found → check account + SSH


Golden rule: eerst SSH remote, dan pas committen.
